# 02 – Versionamiento con Git

- Repositorio: `proyecto_cancelaciones`
- Ramas sugeridas:
  - `main`: versión estable.
  - `dev`: experimentos y desarrollo.
- Buenas prácticas:
  - No subir datos sensibles ni CSV pesados (`data/raw/`, `data/processed/` gitignored).
  - Versionar únicamente scripts, notebooks ligeros y documentación.
  - Commits descriptivos: `feat:`, `fix:`, `refactor:`, `docs:`, etc.
